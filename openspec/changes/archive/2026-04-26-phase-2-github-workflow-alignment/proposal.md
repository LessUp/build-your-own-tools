# Proposal: Phase 2 GitHub Workflow Alignment

## Why

Phase 1 normalized the repository narrative and documentation surface. The next bottleneck is engineering noise:

- workflow inventory is still larger than the close-out posture requires
- several workflows remain more expensive or broader than their current value justifies
- GitHub About metadata is directionally correct but can better reflect the final public story
- the repo's PR and issue workflow should reinforce close-out discipline instead of feature sprawl

## What

### Modified capabilities

- **project** - simplify GitHub automation, align repository metadata, and codify a lightweight OpenSpec-first close-out workflow

### Scope

- review each workflow for keep/remove/manual/scheduled status
- simplify noisy workflow triggers and remove low-value automation
- update PR/issue workflow artifacts where they support the close-out process
- align GitHub repository description, homepage, and topics with the final Pages/README story

### Out of scope

- deep AI role redesign and tooling strategy work reserved for Phase 3
- new release automation features or broad CI expansion
